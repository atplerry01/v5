## 1. Reuse-First Sufficiency Test (leading section)

Per R3.A.6 precedent — before designing new surface, enumerate what existing infrastructure R3.B.1 can reuse.

### 1.1 Default position

R3.B.1 **reuses the existing runtime discipline wherever a parallel exists**. New surface is authorised only where the outbound-effect semantics genuinely differ from internal mutation.

### 1.2 Reusable infrastructure

| Existing primitive | Reuse | New duplicate? |
|---|---|---|
| Postgres-backed polling publisher (`KafkaOutboxPublisher` shape) | **Reuse pattern** — new `OutboundEffectRelay` follows identical claim/poll/retry-shape with different target (adapter vs Kafka producer) | No structural duplicate; reusable `claimed_by` host-partitioning model carries over |
| `IOutbox` / `PostgresOutboxAdapter` — durable row + claim | **NOT reused** (D-R3B-4 ratified: dedicated table). Pattern templates the new table's claim columns | Dedicated table |

| `IIdGenerator` / `IClock` / `IRandomProvider` determinism seams | **Reuse directly** — relay, dispatcher, adapters inject these | No |
| `ICircuitBreaker` + registry | **Reuse directly** — per-provider breakers registered by name `outbound.{providerId}`; grandfathered OPA breaker continues untouched | No |

### 1.3 New surface required (justified)

The following must be built — no existing primitive covers them:

| New primitive | Justification | Layer |

| `OutboundEffectRelay` (concrete) | Background service; claims queue rows, invokes adapter, emits lifecycle events | `src/runtime/outbound-effects/` (relay) + `src/platform/host/adapters/outbound-effects/` (hosted-service shell) |
| `outbound_effect_dispatch_queue` Postgres table + migration | Dedicated per D-R3B-4 | `infrastructure/data/postgres/migrations/` |

| Prohibition guard family `R-OUT-EFF-*` | No existing guard covers "only this seam may execute HTTP" | `claude/guards/runtime.guard.md` |
| Base metrics under `Whycespace.OutboundEffects` meter | Low-cardinality vocabulary per parent §10.2 | Inside `OutboundEffectDispatcher` / relay |

### 1.4 Sufficiency verdict

**SUFFICIENT.** R3.B.1 does not invent new runtime/event/chain/outbox infrastructure. It extends:
- a new aggregate into the existing domain-layer pattern
- a new dispatcher into the existing runtime-layer seam pattern
- a new background relay into the existing host-adapter pattern
- a new table into the existing Postgres infrastructure
- a new guard family into the existing prohibition model

Net-new code lives in its canonical layers; nothing requires a new layer or new middleware stage.

## 2. Ratified constraints guiding every R3.B.1 artefact

Carried verbatim from R3.B §17.2. All five are **design-time invariants** for R3.B.1 — not deferred:

1. **Acknowledged never implies Finalized.** Separate events, separate states, separate adapter result variants. Architecture test asserts they cannot collapse.
2. **Provider operation identity first-class.** `Acknowledged` event carries `{outbound_effect_id, provider_operation_id, correlation_id, causation_id, idempotency_key_used_externally}` as named fields.
3. **Outbound idempotency contract lands at R3.B.1.** Non-null idempotency key required; dedup-on-schedule; retries reuse same key.
4. **Webhook / poll / manual-reconciliation duality designed in R3.B.1.** The aggregate lifecycle supports all three even if only the event surface is defined in this checkpoint.
5. **Prohibition is law on day one.** Guards enforced immediately; whitelist is empty-plus-OPA until R3.B.2.

public sealed record OutboundIdempotencyKey
{
    public string Value { get; }

    public OutboundIdempotencyKey(string value)
    {
        if (string.IsNullOrWhiteSpace(value))
            throw new ArgumentException(OutboundEffectErrors.IdempotencyKeyRequired, nameof(value));

### 4.5 `OutboundAdapterResult` — the six-outcome model (ratified constraint #1)

```csharp
public abstract record OutboundAdapterResult
{
    /// <summary>
    /// 1. Dispatch failed before the provider accepted anything. No provider op id.
    ///    Classification drives retry: Transient → retry; Terminal → stop.
    /// </summary>
    public sealed record DispatchFailedPreAcceptance(
        OutboundAdapterClassification Classification,
        string Reason,
        TimeSpan? RetryAfter = null) : OutboundAdapterResult;

    /// <summary>
    /// 2. Transport-level dispatch succeeded but provider returned no operation id.
    ///    Rare; signals ambiguous state. Triggers reconciliation on finality timeout.
    /// </summary>
    public sealed record DispatchedWithoutProviderOperationId(
        string TransportEvidence) : OutboundAdapterResult;

    /// <summary>
    /// 3. Provider accepted and returned an operation id. Business finality NOT implied.
    /// </summary>
    public sealed record Acknowledged(
        ProviderOperationIdentity ProviderOperation,
        string? AckPayloadDigest = null) : OutboundAdapterResult;

    /// <summary>
    /// 4. Provider returned synchronous terminal success. Business finality achieved
    ///    in-call (rare — only for simple synchronous providers).
    /// </summary>
    public sealed record FinalizedSuccess(
        ProviderOperationIdentity ProviderOperation,
        string FinalityEvidence) : OutboundAdapterResult;

    /// <summary>
    /// 5. Provider returned synchronous terminal failure. Business finality achieved
    ///    (card declined, recipient invalid, etc.). Triggers compensation path.
    /// </summary>
    public sealed record FinalizedFailure(
        ProviderOperationIdentity? ProviderOperation,
        string FailureCode,
        string FailureReason) : OutboundAdapterResult;

    /// <summary>
    /// 6. Provider state unclear. Adapter cannot reliably determine outcome
    ///    (e.g., timeout mid-response on at-most-once adapter). Reconciliation required.
    /// </summary>
    public sealed record ReconciliationRequired(
        OutboundReconciliationCause Cause,
        string DiagnosticEvidence) : OutboundAdapterResult;
}

public sealed record ProviderOperationIdentity(
    string ProviderId,                  // duplicated for safety across system boundaries
    string ProviderOperationId,         // opaque provider-assigned identifier
    string? IdempotencyKeyUsed = null); // the key we actually sent (may differ in format from local key)
```

Six cases exactly. Any adapter attempting to collapse (e.g., returning `Acknowledged` when finality was actually achieved) is a guard violation caught via unit-level adapter tests and/or architecture tests on the adapter's return statements (R-OUT-EFF-FINALITY-01).

## 5. Aggregate + Lifecycle Events

### 5.1 `OutboundEffectStatus` (append-only enum, ordinal discipline)

```csharp
public enum OutboundEffectStatus
{
    Scheduled = 0,
    Dispatched = 1,
    Acknowledged = 2,
    Finalized = 3,                 // terminal
    Cancelled = 4,                 // terminal
    TransientFailed = 5,           // transient; returns to Scheduled on retry
    RetryExhausted = 6,            // terminal unless compensation transitions
    ReconciliationRequired = 7,
    Reconciled = 8,                // terminal
    CompensationRequested = 9,     // transient; companion state during compensation workflow
}
```

### 5.2 `OutboundFinalityOutcome` (on Finalized / Reconciled)

```csharp
public enum OutboundFinalityOutcome
{
    Succeeded,
    BusinessFailed,
    PartiallyCompleted,
    ManualIntervention,    // reconciliation could not resolve; human-owned
}
```

### 5.3 Lifecycle events (eleven; one per non-trivial state transition)

Each event carries `AggregateId`, `CorrelationId`, `CausationId` implicitly (domain `AggregateRoot` + runtime envelope), plus the named fields below:

| Event | Fields beyond envelope |
|---|---|
| `OutboundEffectScheduledEvent` | `ProviderId`, `EffectType`, `IdempotencyKey`, `PayloadTypeDiscriminator`, `SchedulerActorId`, `DispatchTimeoutMs`, `TotalBudgetMs`, `AckTimeoutMs`, `FinalityWindowMs`, `MaxAttempts` |
| `OutboundEffectDispatchedEvent` | `AttemptNumber`, `DispatchStartedAt`, `DispatchCompletedAt`, `TransportEvidenceDigest?` |
| `OutboundEffectAcknowledgedEvent` | `ProviderOperationIdentity` (ProviderId + ProviderOperationId + IdempotencyKeyUsed), `AckPayloadDigest?` — satisfies ratified constraint #2 |
| `OutboundEffectDispatchFailedEvent` | `AttemptNumber`, `Classification (Transient\|Terminal\|Ambiguous)`, `Reason`, `RetryAfterMs?` |
| `OutboundEffectRetryAttemptedEvent` | `AttemptNumber`, `NextAttemptAt`, `BackoffMs`, `PrecedingClassification` |
| `OutboundEffectRetryExhaustedEvent` | `TotalAttempts`, `FinalClassification` |
| `OutboundEffectFinalizedEvent` | `FinalityOutcome`, `FinalityEvidenceDigest`, `FinalizedAt`, `FinalitySource (Push\|Poll\|SynchronousAck)` |
| `OutboundEffectReconciliationRequiredEvent` | `Cause (AckTimeout\|FinalityTimeout\|ProviderDisagreement\|DispatchAmbiguous)`, `ObservedAt` |
| `OutboundEffectReconciledEvent` | `Outcome (Succeeded\|BusinessFailed\|PartiallyCompleted\|ManualIntervention)`, `EvidenceDigest`, `ReconcilerActorId` |
| `OutboundEffectCompensationRequestedEvent` | `TriggeringOutcome (BusinessFailed\|PartialCompensated)`, `OwnerAggregateRef (type, id)` |
| `OutboundEffectCancelledEvent` | `CancellationReason`, `PreDispatch (true \| false)` |

### 5.4 Aggregate invariants

```
start              Start() → emits Scheduled
Scheduled          → Dispatched (emits Dispatched)
                   → Cancelled (emits Cancelled, pre-dispatch)
                   → DispatchFailedPreAcceptance (emits DispatchFailedEvent + TransientFailed/Terminal)
Dispatched         → Acknowledged (emits Acknowledged; REQUIRES ProviderOperationIdentity non-null)
                   → Finalized (emits Finalized; SynchronousAck source; rare — only when adapter returns FinalizedSuccess/Failure)
                   → ReconciliationRequired (if DispatchedWithoutProviderOperationId and AckTimeout expires)
                   → DispatchFailed (ambiguous; triggers ReconciliationRequired for AtMostOnce adapters)
Acknowledged       → Finalized (Push or Poll source)
                   → ReconciliationRequired (FinalityTimeout expired, ProviderDisagreement detected)
TransientFailed    → Scheduled (retry pending; emits RetryAttemptedEvent on re-queue)
                   → RetryExhausted (emits RetryExhausted)
RetryExhausted     → CompensationRequested (emits CompensationRequested)
                   → terminal (if no compensation registered — operator alert)
ReconciliationRequired → Reconciled (emits Reconciled with outcome)
Finalized          → CompensationRequested (only if outcome ∈ {BusinessFailed, PartiallyCompleted} AND compensation registered)
                   → terminal otherwise
Reconciled         → CompensationRequested (same condition)
                   → terminal otherwise
Cancelled          → terminal
```

R3.B.1 implements the full state machine in `Apply` but only the Scheduled / Dispatched / Acknowledged / DispatchFailed / RetryAttempted / RetryExhausted / Cancelled transitions are emitted by the relay — Finalized / Reconciliation / Compensation come online in R3.B.4 / R3.B.5. The event records and aggregate handlers exist at R3.B.1 so the shape cannot drift between checkpoints.

### 5.5 Replay discipline

- Payload is typed via `PayloadTypeDiscriminator` — reuses the workflow-payload typed-registry pattern.
- Evidence digests (`AckPayloadDigest`, `FinalityEvidenceDigest`, `EvidenceDigest`) are SHA-256 of the source payload; full payloads are NOT stored on the event stream (privacy + size). The raw payload lives on the dispatch queue or in the provider response audit log, not the event store.
- All other fields are replay-stable.

## 6. Queue Table Schema (per D-R3B-4)

```sql
-- infrastructure/data/postgres/migrations/003_create_outbound_effect_dispatch_queue.sql

CREATE TABLE outbound_effect_dispatch_queue (
    effect_id         UUID PRIMARY KEY,                  -- matches OutboundEffectAggregate id
    provider_id       TEXT NOT NULL,
    effect_type       TEXT NOT NULL,
    idempotency_key   TEXT NOT NULL,
    status            TEXT NOT NULL,                     -- Scheduled | Dispatched | Acknowledged | ...
    attempt_count     INT  NOT NULL DEFAULT 0,
    max_attempts      INT  NOT NULL,
    next_attempt_at   TIMESTAMPTZ NOT NULL,
    dispatch_deadline TIMESTAMPTZ NOT NULL,              -- = created_at + total_budget_ms
    ack_deadline      TIMESTAMPTZ NULL,                  -- set on Dispatched transition
    finality_deadline TIMESTAMPTZ NULL,                  -- set on Acknowledged transition
    last_error        TEXT NULL,
    claimed_by        TEXT NULL,                         -- host id (null = unclaimed)
    claimed_at        TIMESTAMPTZ NULL,
    created_at        TIMESTAMPTZ NOT NULL,
    updated_at        TIMESTAMPTZ NOT NULL,

    CONSTRAINT ux_outbound_effect_dispatch_queue_idempotency
        UNIQUE (provider_id, idempotency_key)            -- enforces dedup-on-schedule at DB level
);

CREATE INDEX ix_outbound_effect_dispatch_queue_next_attempt
    ON outbound_effect_dispatch_queue (status, next_attempt_at)
    WHERE claimed_by IS NULL AND status IN ('Scheduled', 'TransientFailed');

CREATE INDEX ix_outbound_effect_dispatch_queue_ack_deadline
    ON outbound_effect_dispatch_queue (ack_deadline)
    WHERE ack_deadline IS NOT NULL AND status = 'Dispatched';

CREATE INDEX ix_outbound_effect_dispatch_queue_finality_deadline
    ON outbound_effect_dispatch_queue (finality_deadline)
    WHERE finality_deadline IS NOT NULL AND status = 'Acknowledged';
```

The queue is a **projection** of the event stream — dedup uniqueness at DB level is belt-and-suspenders with the aggregate-level dedup at §7.

### Rules

- **R-OUT-EFF-QUEUE-01:** Queue row is inserted in the same transaction as the `OutboundEffectScheduledEvent` persist. Atomicity via the existing `EventStoreService` transaction.
- **R-OUT-EFF-QUEUE-02:** Queue row is a projection; lost/recreated queue rows MUST be reconstructable from the event stream (operator command for recovery — R3.B.3 scope).
- **R-OUT-EFF-QUEUE-03:** `claimed_by` is set atomically via single-row UPDATE WHERE claimed_by IS NULL; identical shape to the existing outbox claim.

## 7. Dispatcher Implementation Outline

```csharp
public sealed class OutboundEffectDispatcher : IOutboundEffectDispatcher
{
    // Injected:
    //   IEventStore           _eventStore;
    //   IOutboundEffectQueueStore _queueStore;  // thin wrapper over Postgres queue table
    //   IIdGenerator          _idGenerator;
    //   IClock                _clock;
    //   IOutboundEffectOptionsRegistry _optionsRegistry;   // per-provider options
    //   IPayloadTypeRegistry  _payloadTypes;
    //   OutboundEffectLifecycleEventFactory _lifecycleFactory;  (from T2E)

    public async Task<OutboundEffectScheduleResult> ScheduleAsync(
        OutboundEffectIntent intent, CommandContext context, CancellationToken ct)
    {
        // 1. Validate: provider registered, payload type registered, idempotency key non-null.
        // 2. Dedup check: if (providerId, idempotencyKey) already exists on queue → return existing id + DedupHit=true.
        // 3. Resolve options: OutboundEffectOptions merged with per-intent overrides.
        // 4. Construct OutboundEffectAggregate via lifecycle factory.Start(...).
        //    Aggregate emits OutboundEffectScheduledEvent.
        // 5. Persist the event via canonical runtime persist → chain → outbox pipeline.
        // 6. Insert queue row in same transaction (via IEventStore's transaction scope or
        //    dedicated unit-of-work — to be designed in §7 D-R3B1-1 below).
        // 7. Return OutboundEffectScheduleResult(effectId, DedupHit=false).
    }

    public async Task<bool> CancelScheduledAsync(
        OutboundEffectId effectId, string reason, CommandContext context, CancellationToken ct)
    {
        // 1. Load aggregate via event-store replay (NOT via queue row).
        // 2. Assert Status == Scheduled. If not, return false (idempotent no-op).
        // 3. Construct Cancelled event via factory; persist via pipeline.
        // 4. Update queue row status to Cancelled.
        // 5. Return true.
    }
}
```

### Key design notes

- Dispatcher does NOT invoke adapters. That is the relay's sole role.
- Dispatcher runs within the canonical 11-stage pipeline when called via command dispatch; it MAY also be called directly from domain-event integration handlers (e.g., `PayoutCompletedEvent → schedule notification`). In both cases, `CommandContext` is available for actor/correlation propagation.
- Payload payload size cap: 256 KB at R3.B.1 (provider-specific overrides in R3.B.2+).

## 8. Relay + Adapter Skeleton

### 8.1 `OutboundEffectRelay` (background service)

```csharp
public sealed class OutboundEffectRelay : BackgroundService
{
    // Injected: IOutboundEffectQueueStore, IOutboundEffectAdapterRegistry,
    //           OutboundEffectLifecycleEventFactory, IEventStoreService,
    //           ICircuitBreakerRegistry, IClock, IRandomProvider, ILogger, Meter

    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        // Loop:
        //   1. Claim a batch of rows WHERE status ∈ {Scheduled, TransientFailed}
        //      AND next_attempt_at <= now AND claimed_by IS NULL.
        //      Mark claimed_by = this.HostId atomically.
        //   2. For each claimed row in parallel (bounded concurrency):
        //        a. Resolve IOutboundEffectAdapter by provider_id.
        //        b. Get per-provider ICircuitBreaker from registry.
        //        c. Build OutboundEffectDispatchContext.
        //        d. Start linked CTS bounded by DispatchTimeoutMs.
        //        e. Call breaker.ExecuteAsync(() => adapter.DispatchAsync(ctx, linkedCt)).
        //        f. Translate OutboundAdapterResult into lifecycle event via factory.
        //        g. Persist event through canonical pipeline.
        //        h. Update queue row: new status, attempt_count++, next_attempt_at,
        //           ack_deadline/finality_deadline as applicable, claimed_by = null.
        //        i. Emit metrics.
        //   3. Sleep until next poll cycle.
        // On host shutdown: drain in-flight; unclaimed rows revert automatically
        // (claimed_by stays NULL because claim + dispatch are atomic units of work).
    }
}
```

R3.B.1 ships the relay skeleton with correct structural shape. **Retry classification + backoff refinement are R3.B.3 scope** — at R3.B.1 the relay implements basic retry (fixed backoff from options; exhaustion after MaxAttempts) and basic classification (Transient / Terminal / Ambiguous mapped from `OutboundAdapterResult` variants).

## 10. Prohibition Guard Set (law from day R3.B.1 lands)

### 10.1 Core prohibition

- **`R-OUT-EFF-PROHIBITION-01` (S0):** `HttpClient`, `HttpMessageInvoker`, `HttpClientHandler`, `WebClient`, `SocketsHttpHandler` usage is FORBIDDEN except in:
  (a) `src/platform/host/adapters/Opa*.cs` — OPA internal-policy family, **grandfathered per D-R3B-10**, with explicit R3.B ratification-document reference;
  (b) `src/platform/host/adapters/outbound-effects/**/*Adapter.cs` — new production adapters;
  (c) `src/runtime/outbound-effects/**` — relay / dispatcher plumbing (no direct HTTP; adapter delegation only — see R-OUT-EFF-PROHIBITION-03);
  (d) `src/shared/contracts/infrastructure/**` — contract/type definitions referencing HTTP types for interfaces (not executing calls).
  CI grep: any `new HttpClient` / injected `HttpClient` outside the whitelist → S0 fail.

- **`R-OUT-EFF-PROHIBITION-02` (S0):** Third-party SDK namespaces (`Stripe.*`, `Twilio.*`, `SendGrid.*`, `Slack.*`, `Amazon.*` where it references service SDKs, `Microsoft.Azure.*` where it references Azure service SDKs, `Google.Cloud.*`, etc.) are FORBIDDEN in `src/domain/**`, `src/engines/**`, `src/runtime/**` (except `src/runtime/outbound-effects/**` — and even there, only as delegation targets to adapters, not direct references), `src/projections/**`, `src/systems/**`. Production SDK references MUST live ONLY in `src/platform/host/adapters/outbound-effects/{Provider}Adapter.cs`.

- **`R-OUT-EFF-PROHIBITION-03` (S0):** Only `OutboundEffectRelay` may invoke `IOutboundEffectAdapter.DispatchAsync`. No other caller. CI grep asserts zero matches for `adapter.DispatchAsync(` or `IOutboundEffectAdapter.*DispatchAsync` outside the relay.

- **`R-OUT-EFF-PROHIBITION-04` (S1):** `src/domain/**` may not call `IOutboundEffectDispatcher.ScheduleAsync` directly — scheduling must go through engines or runtime control plane (parallel to the existing domain-purity rules).

- **`R-OUT-EFF-PROHIBITION-05` (S1):** Test-scoped adapters (`NoOpOutboundEffectAdapter` and any future `test/` subtree members) MUST be registered only in `TestHost` / `RealInfraTestHost` composition — NOT in production composition. Architecture test asserts `NoOpOutboundEffectAdapter` is not reachable from `Program.cs` composition.

### 10.2 Seam integrity

- **`R-OUT-EFF-SEAM-01` (S1):** `OutboundEffectAggregate.Start(...)` is non-public outside `Whycespace.Engines.T2E.OutboundEffects.Lifecycle.OutboundEffectLifecycleEventFactory`. Construction from other callers is FORBIDDEN. Enforced via accessibility + architecture test.

- **`R-OUT-EFF-SEAM-02` (S1):** `IOutboundEffectAdapter` implementations MUST be stateless singletons. No mutable instance fields. Architecture test pins.

- **`R-OUT-EFF-SEAM-03` (S1):** Every `OutboundEffect*Event` MUST be emitted via `OutboundEffectLifecycleEventFactory`. Direct construction (`new OutboundEffectDispatchedEvent(...)`) from relay / dispatcher / projection / adapters is FORBIDDEN (parallel to `R-WORKFLOW-CANCELLATION-FACTORY-01`).

### 10.3 Outcome-model discipline (ratified constraint #1)

- **`R-OUT-EFF-FINALITY-01` (S1):** Adapter `DispatchAsync` MUST return one of the six `OutboundAdapterResult` variants. Architecture test scans adapter implementations for returns unreachable from the sealed type hierarchy.

- **`R-OUT-EFF-FINALITY-02` (S1):** `Acknowledged` MUST carry a non-null `ProviderOperationIdentity`. `FinalizedSuccess` MUST carry a non-null `ProviderOperationIdentity` unless provider genuinely does not assign one (documented adapter exception). Aggregate `Apply(OutboundEffectAcknowledgedEvent)` asserts this.

### 10.4 Idempotency (ratified constraint #3)

- **`R-OUT-EFF-IDEM-01` (S1):** `OutboundEffectIntent.IdempotencyKey` is REQUIRED. Grep: every `OutboundEffectIntent` construction has non-null key. Enforced by constructor validation — cannot be bypassed.

- **`R-OUT-EFF-IDEM-02` (S1):** Dispatcher schedules are deduped by `(provider_id, idempotency_key)` at both aggregate-level and DB-level (unique constraint on queue table). Duplicate schedule returns existing `OutboundEffectId` with `DedupHit=true` and emits `outbound.effect.schedule.dedup_hit` metric.

- **`R-OUT-EFF-IDEM-03` (S1):** For `ProviderIdempotent` adapters, every outbound HTTP call MUST propagate the idempotency key to the provider (e.g., `Idempotency-Key: {key}` header). Grep: adapter file's `DispatchAsync` references `ctx.IdempotencyKey` in its request construction.

- **`R-OUT-EFF-IDEM-04` (S1):** Retries reuse the SAME `effect_id` as the idempotency key payload — regenerating is FORBIDDEN. Relay injects the stable `effect_id` on every retry.

- **`R-OUT-EFF-IDEM-05` (S1):** `AtMostOnceRequired` adapters must NOT retry after `Dispatched` transition lands — any post-dispatch transient failure must translate to `ReconciliationRequired`, not retry. Aggregate `Apply` enforces via classification guard.

### 10.5 Timeout / cancellation (ratified; R3.B.1 scope)

- **`R-OUT-EFF-TIMEOUT-01` (S1):** Adapter `DispatchAsync` MUST consume the `CancellationToken` it receives. Architecture test: every adapter file has the token parameter flow into at least one downstream call.

- **`R-OUT-EFF-TIMEOUT-02` (S1):** Per-provider `OutboundEffectOptions.{DispatchTimeoutMs, TotalBudgetMs, AckTimeoutMs, FinalityWindowMs, MaxAttempts}` are REQUIRED at composition; no hardcoded defaults. Composition root throws on missing config (parallel to R-CFG-R3).

### 10.6 Observability (ratified; R3.B.1 ships the meter)

- **`R-OUT-EFF-OBS-01` (S2):** All outbound-effect metrics live on the `Whycespace.OutboundEffects` meter. No shadow meters per provider.

- **`R-OUT-EFF-OBS-02` (S2):** Outcome tag vocabulary is locked per §9. New values require guard-level promotion.

### 10.7 Determinism

- **`R-OUT-EFF-DET-01` (S1):** Dispatcher, relay, and all adapters MUST use `IIdGenerator` / `IClock` / `IRandomProvider` only. `Guid.NewGuid()`, `DateTime.UtcNow`, `DateTimeOffset.UtcNow`, `Random.*` are FORBIDDEN in `src/runtime/outbound-effects/**`, `src/engines/T2E/outbound-effects/**`, `src/platform/host/adapters/outbound-effects/**`. Architecture test pins.

- **`R-OUT-EFF-DET-02` (S2):** Adapter classification mapping is a pure function of `(provider-response shape, classification-rules)`. No wall-clock / random inputs.

### 10.8 Placement
