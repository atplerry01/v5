# Content-System Extraction — new rule IDs and migration backlog

This capture covers two things:

1. **Promotable guard rules** pinned by

2. **Deferred Category A migrations**: Content Inventory sites that remain on

## Promotable guard rules

### CS-STRUCTURAL-OWNER-01 (S0)

`DocumentAggregate.Create` MUST reject an empty `StructuralOwnerRef`. Content
cannot exist without a structural parent (cluster / sub-cluster). Business
aggregates may only reference content via typed `ContentRef` — they never
act as structural owners.

**Test:** `DocumentAggregate_Create_rejects_empty_StructuralOwnerRef`.

### CS-LIFECYCLE-CANONICAL-01 (S1)

`DocumentStatus` MUST declare all four canonical lifecycle states:
`Draft`, `Active`, `Archived`, `Superseded`. Removing any is a lifecycle-model
regression.

**Test:** `DocumentStatus_enum_declares_the_four_canonical_lifecycle_states`.

### CS-EVENT-MODEL-01 (S1)

The canonical content event set MUST be present in the
`Whycespace.Domain.ContentSystem.Document.CoreObject.Document` namespace:
`DocumentCreatedEvent`, `DocumentVersionAttachedEvent`,
`DocumentSupersededEvent`, `DocumentActivatedEvent`, `DocumentArchivedEvent`
(+ existing `DocumentRestoredEvent`, `DocumentMetadataUpdatedEvent`).

**Test:** `Canonical_content_events_are_declared`.

### CS-DECOUPLE-ORDER-01 (S1)

`OrderAggregate` MUST NOT declare any embedded `string Description|Body|
Notes|Comment` property. Description prose is externalised to a
`DocumentAggregate`; `OrderAggregate` exposes a typed `ContentRef`
property.

**Test:** `OrderAggregate_references_content_via_ContentRef_not_prose_string`.

### CS-DOC-PROSE-LOCK-01 (S1)

`DocumentAggregate` (the content-aggregate root) MUST NOT declare a
`string Body|Content|Payload|Description|Notes|Comment|Html|Markdown|
Abstract|Blurb` property. The root owns identity, metadata, and refs; the
content bytes live on the version / file artifact.

**Test:** `DocumentAggregate_declares_no_embedded_prose_content_fields`.

## Deferred Category A migration backlog

The following prose-bearing aggregates were inventoried in Phase 1 but not
yet decoupled. Each needs its own per-aggregate migration pass matching the
proof pattern established by `OrderAggregate`:

| # | Aggregate / Entity | Field(s) | Owning system |
|---|---|---|---|
| 1 | `EnforcementRuleAggregate.Description` | prose narrative | economic-system |
| 2 | `ViolationAggregate.Reason` | prose reason | economic-system |
| 3 | `SettlementAggregate.FailureReason` | prose narrative | economic-system |
| 4 | `EvidenceSummary.Value` (user-flagged EXTERNALISE) | summary prose | economic-system |
| 5 | `KanbanCard.Title` + `KanbanCard.Description` | user-editable prose | operational-system |
| 6 | `KanbanList.Name` | list label (prose-capable) | operational-system |
| 7 | `KanbanAggregate.Name` | board label (prose-capable) | operational-system |
| 8 | `CounterpartyProfile.Name` | party display name | business-system |
| 9 | Event payloads mirroring the above (`EnforcementRuleDefinedEvent.Description`, `KanbanCard*Event.Title/Description`, `TodoCreatedEvent.Title`, `TodoTitleRevisedEvent.Title`) | prose parameters | various |

**Per-row migration pattern (reference: `OrderAggregate`):**
1. Create local `ContentRef` VO in the aggregate's `value-object/` folder.
2. Replace `string *Prose` property with `ContentRef *` property.
3. Update the `*CreatedEvent` / `*DetailRevised*Event` signature to carry
   `ContentRef` in place of the prose parameter.
4. Update the aggregate's `EnsureInvariants` to assert
   `<NewContentRef> != default`.
5. Update any projection / reducer / test that constructs the aggregate or
   its events.
6. Extend `ContentSystemEnforcementLockTests` with a per-aggregate decouple
   test mirroring `OrderAggregate_references_content_via_ContentRef_not_prose_string`.

**Fields explicitly retained per user directive (do not migrate):**
- `EnforcementRuleAggregate.RuleName`
- `ExpenseLine.Description`
- Category B (VO-wrapped reasons) and Category C (event audit reasons).

## Notes on the ContentAggregate integration decision

Per user decisions (2026-04-21):
- `DocumentAggregate` (content-system/document/core-object/document) is the
  canonical `ContentAggregate` — no parallel `content-system/content` context
  was created.
- `DocumentVersionAggregate` (content-system/document/lifecycle-change/version)
  is the canonical `ContentVersion`, reused as-is.
- `StructuralOwnerRef` is mandatory; `BusinessOwnerRef` (polymorphic by
  `BusinessOwnerKind`) is optional.
- No events were retired or renamed; two new events
  (`DocumentVersionAttachedEvent`, `DocumentSupersededEvent`) were added and
  `DocumentCreatedEvent` was extended with the new owner refs (pre-production
  schema; no wire-compat break to defend).
