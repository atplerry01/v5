# New Rules Capture — 2026-04-08 — Guards

## Issue Summary

Phase 1.5 §5.1.2 Step B (BPV-D01) discovered that the host→domain
prohibition encoded in `DG-R5-HOST-DOMAIN-FORBIDDEN`
(claude/guards/dependency-graph.guard.md) and `R-DOM-01`
(claude/guards/runtime.guard.md) had two independent failure modes:

1. **Mechanical predicate hole.** Both rules grep only for
   `using Whycespace.Domain.*` lines. Eleven live binding sites in
   `src/platform/host/composition/**` bypassed the predicate by writing
   the bindings as fully-qualified type expressions
   (`typeof(Whycespace.Domain.X.Y)`, `(Whycespace.Domain.X.Y)e`) and as
   namespace aliases (`using DomainEvents = Whycespace.Domain.X.Y`).
   §5.1.1 PASS verification missed every one of these sites.

2. **Folklore-vs-canon contradiction.** The comment block in
   `src/platform/host/composition/constitutional/policy/ConstitutionalPolicyBootstrap.cs`
   (pre-remediation lines 17-18) asserted that
   `src/platform/host/composition/**` was a *“canonical exempt zone for
   domain-typed wiring (per rule 11.R-DOM-01)”*. The body of R-DOM-01
   in `claude/guards/runtime.guard.md` contains **no such exemption**;
   the EXEMPT PATHS list at the rule definition does not include
   `src/platform/host/composition/**`. The only nod to a
   composition-root exemption was a STATUS line at the bottom of the
   rule admitting that `grep -R "Todo" src/platform/host/` returned
   matches only inside `composition/**` — described as “exempt by
   design” without a corresponding rule clause. The implementation
   relied on this informal status admission; the rule body never
   formalized it.

| Source | Stated Position | File |

| Implementation comment | Composition is exempt under R-DOM-01 | `src/platform/host/composition/constitutional/policy/ConstitutionalPolicyBootstrap.cs` (pre-remediation) |

| DG-R5-HOST-DOMAIN-FORBIDDEN | Host **may not** contain `using Whycespace.Domain.*` (no composition carve-out) | `claude/guards/dependency-graph.guard.md` (lines 217-228, pre-Step C-G) |

The footer’s informal admission contradicted both its own rule body
and the dependency-graph guard. The implementation comment took the
footer side; the canonical reading takes the rule-body side.

## Corrected Canonical Interpretation

`src/platform/host/composition/**` is **not** exempt from the host→domain
prohibition. The previously-tolerated typed domain references inside
composition modules were a latent S1 violation, not a canonical carve-out.
After Phase 1.5 §5.1.2 Step C remediation, composition modules contain
zero typed `Whycespace.Domain.*` references. The R-DOM-01 footer has
been rewritten to remove the “exempt by design” admission and to
explicitly list `src/runtime/event-fabric/domain-schemas/**` as the
**only** runtime-side location permitted to hold typed domain
references — the canonical schema-binding seam introduced by BPV-D01
remediation.

The mechanical predicates of both rules have been strengthened to
match three forms instead of one:

| Form | Example | Pre-Step C-G | Post-Step C-G |
|---|---|---|---|
| `using` directive | `using Whycespace.Domain.X.Y;` | caught | caught |
| Fully-qualified expression | `typeof(Whycespace.Domain.X.Y)` | **bypass** | **caught** |
| Namespace alias | `using DomainEvents = Whycespace.Domain.X.Y;` | **bypass** | **caught** |

Comment lines (lines whose first non-whitespace characters are `//` or
`*`) are excluded from the strengthened predicate so that the canonical
intent-comment in
`src/platform/host/composition/runtime/RuntimeComposition.cs:80`
(*“runtime middleware cannot reference Whycespace.Domain.*”*) is
preserved as documentation, not flagged as a violation.

## Enforcement Consequence

- `scripts/dependency-check.sh` now scans `src/platform/host/**` for
  any non-comment occurrence of `Whycespace.Domain.` and reports each
  match under `DG-R5-HOST-DOMAIN-FORBIDDEN (fully-qualified or alias)`.
  Any new bypass attempt fails the script with `Status: FAIL`.
- `claude/guards/dependency-graph.guard.md` DG-R5-HOST-DOMAIN-FORBIDDEN
  now enumerates clauses 1–5 covering csproj reference, `using`,
  fully-qualified expression, namespace alias, and SharedKernel
  primitive — the prior 1–3 form is superseded.
- `claude/guards/runtime.guard.md` R-DOM-01 now lists the three forbidden
  forms explicitly, removes the implicit composition exemption, and
  adds `src/runtime/event-fabric/domain-schemas/**` to the EXEMPT PATHS
  list with an authority pointer back to BPV-D01 remediation.
- Severity: DG-R5-HOST-DOMAIN-FORBIDDEN remains **S0** (script-fail).
  R-DOM-01 remains **S1** (CI-fail). Neither was weakened.
- Verification (this Step C-G pass): `bash scripts/dependency-check.sh`
  → `Violations: 0`, `Status: PASS` with the strengthened predicate
  active.

## Classification, Source, Description, Proposed Rule, Severity

- **CLASSIFICATION:** governance / boundary-control / guards
- **SOURCE:** Phase 1.5 §5.1.2 Step B (BPV-D01) and Step C-G
  (this capture). Discovered while exhaustively probing
  `src/platform/host/**` for domain leakage on 2026-04-08.
- **DESCRIPTION:** The host→domain prohibition was encoded with a
  `using`-line predicate that fully-qualified type expressions and
  namespace aliases bypassed; eleven such bypass sites survived
  §5.1.1 PASS unnoticed. A second, related drift was an implementation
  comment claiming a `composition/**` exemption that the canonical
  rule body never granted.
- **PROPOSED RULE (now applied):**
  1. `DG-R5-HOST-DOMAIN-FORBIDDEN` and `R-DOM-01` predicates MUST
     match three forms: `using` directive, fully-qualified expression,
     and namespace alias. Comment lines are excluded.
  2. `src/platform/host/composition/**` is NOT exempt from
     host→domain prohibition.
  3. `src/runtime/event-fabric/domain-schemas/**` IS the canonical
     and exclusive runtime-side seam for typed domain references in
     schema binding; it MUST be listed in the R-DOM-01 EXEMPT PATHS.
  4. `scripts/dependency-check.sh` MUST mechanically enforce form (1)
     for `src/platform/host/**`.
- **SEVERITY:** S0 for clause 1 mechanical bypass closure (because
  the bypassed rule was itself S0); S1 for the folklore comment
  contradiction (documentation drift, not behavior).
