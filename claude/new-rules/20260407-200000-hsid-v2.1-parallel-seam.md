## DESCRIPTION

The HSID v2.1 implementation introduces a SECOND deterministic identity seam,
`Whyce.Shared.Kernel.Determinism.IDeterministicIdEngine`, alongside the existing
canonical seam `Whyce.Shared.Kernel.Domain.IIdGenerator`
(`src/platform/host/composition/core/DeterministicIdGenerator.cs`).

The existing `determinism.guard.md` declares `IIdGenerator.Generate(seed)` as
the SINGLE permitted ID source and `DeterministicIdGenerator` as the SINGLE
exception class. The HSID v2.1 seam violates that single-seam invariant by
design — it produces topology-encoded, bucket-scoped, sequenced compact IDs
that cannot be expressed through the `Guid Generate(string seed)` signature.

## SCOPE OF THE TWO SEAMS

| Seam | Purpose | Output | Source of truth |
|------|---------|--------|-----------------|
| `IIdGenerator` | Internal Guid derivation for adapters, hashes, row ids | `Guid` | `DeterministicIdGenerator` (SHA256 of seed → Guid) |
| `IDeterministicIdEngine` | External-facing compact correlation IDs | `string` `PPP-LLLL-TTT-TOPOLOGY-SEQ` | `Whyce.Engines.T0U.Determinism.DeterministicIdEngine` |

`Environment.Tick*`. The HSID seam additionally forbids unbucketed sequences,
domain-layer generation, and topology-less IDs.
