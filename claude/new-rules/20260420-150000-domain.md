# Drift capture: Aggregate identity + invariant enforcement gaps

## CLASSIFICATION
domain / identity-topology-enforcement

## SOURCE
Executed from `claude/project-prompts/20260420-150000-runtime-structural-system-enforcement.md`. Scan performed against `src/domain/**` on branch `dev_wip`.

## DESCRIPTION

### Finding 1 — Raw `Guid` relationship FKs on aggregate roots bypass VO discipline

Sixteen aggregate-root sites declare relationship foreign keys as `public Guid <Name>Id { get; private set; }` instead of wrapping in a typed value-object or a canonical `Ref` VO. Primary aggregate IDs are already typed VOs (e.g., `VaultId`, `TransactionId`) — the drift is on inter-aggregate references.

Affected sites (enumerated from Grep of `src/domain/**/*Aggregate.cs`):

- `economic-system/capital/vault/aggregate/VaultAggregate.cs` — `OwnerId`
- `economic-system/transaction/wallet/aggregate/WalletAggregate.cs` — `OwnerId`, `AccountId`
- `economic-system/capital/reserve/aggregate/ReserveAggregate.cs` — `AccountId`
- `economic-system/ledger/obligation/aggregate/ObligationAggregate.cs` — `CounterpartyId`
- `economic-system/ledger/entry/aggregate/LedgerEntryAggregate.cs` — `JournalId`, `AccountId`
- `economic-system/transaction/charge/aggregate/ChargeAggregate.cs` — `TransactionId`
- `economic-system/capital/allocation/aggregate/CapitalAllocationAggregate.cs` — `SourceAccountId`
- `economic-system/capital/binding/aggregate/BindingAggregate.cs` — `AccountId`, `OwnerId`
- `economic-system/capital/asset/aggregate/AssetAggregate.cs` — `OwnerId`
- `economic-system/transaction/limit/aggregate/LimitAggregate.cs` — `AccountId`
- `economic-system/transaction/instruction/aggregate/InstructionAggregate.cs` — `FromAccountId`, `ToAccountId`
- `economic-system/revenue/pricing/aggregate/PricingAggregate.cs` — `ContractId`

An `OwnerId` VO already exists at `src/domain/economic-system/capital/account/value-object/OwnerId.cs` — some sites have a VO available but use raw `Guid`; other refs (`AccountRef`, `TransactionRef`, `ContractRef`) would need to be introduced.

### Finding 2 — Empty `EnsureInvariants()` blocks in aggregate roots (REMEDIATED IN THIS PASS)

Two aggregates held doc-only `EnsureInvariants()` bodies that allowed invariant violations to pass through `RaiseDomainEvent`:

- `src/domain/operational-system/sandbox/todo/aggregate/TodoAggregate.cs`
- `src/domain/operational-system/sandbox/kanban/aggregate/KanbanAggregate.cs`

Both were remediated in this pass to assert non-empty identity + required name/title invariants. Error codes added: `TodoErrors.EmptyIdentity`, `KanbanDomainErrors.EmptyBoardIdentity`.

## PROPOSED_RULE

### Proposed rule: D-ID-REF-01 — Inter-aggregate references MUST be typed value objects

Every inter-aggregate reference property on an aggregate root (or entity) MUST be a typed value object — either an owning aggregate's ID VO or a canonical `*Ref` VO. Raw `System.Guid` (or `string`) as an inter-aggregate reference field is FORBIDDEN. Rationale: primitive references silently defeat the structural-parent discipline enforced elsewhere (D69/D74 for audit/violation); a zero-Guid relationship FK is structurally invalid yet passes compile. Enforcement: grep `src/domain/**/*Aggregate.cs` for `public\s+Guid\s+\w+Id\s*\{\s*get` — zero hits required.

Severity: S1 (architectural).

### Proposed rule: D-INV-NON-EMPTY-01 — Aggregate `EnsureInvariants()` MUST enforce identity non-emptiness

Every `AggregateRoot` subclass MUST have a non-empty `EnsureInvariants()` body that, at minimum, asserts its primary identity is non-empty post-Apply. A body containing only a comment is treated as absent. Rationale: `AggregateRoot.RaiseDomainEvent` already runs `EnsureInvariants()` after `Apply()`; empty body = silent invariant drift. Enforcement: AST scan rejecting `EnsureInvariants` bodies that contain only trivia/comments.

Severity: S2 (structural).

## SEVERITY
S1 (D-ID-REF-01) + S2 (D-INV-NON-EMPTY-01).
