# NEW RULE — ALL MUTATING ENDPOINTS REQUIRE AUTH

CLASSIFICATION: governance / enforcement / guards
SOURCE: E2E lifecycle validation (todo update negative test), 2026-04-07
SEVERITY: S1

## DESCRIPTION
Any HTTP endpoint that mutates state MUST require an authenticated identity before any business logic executes.

Observed violation:
- `POST /api/todo/update` accepted a request without an `Authorization` header
- Request was rejected only by the idempotency middleware ("Duplicate command detected"), not by auth
- A first-time unauthorized update would have succeeded
- Violates $8 (ALL operations through WHYCEPOLICY)

## PROPOSED_RULE
All `POST`, `PUT`, `PATCH`, `DELETE` endpoints MUST be protected by EITHER:
- `[Authorize]` attribute on the controller/action, OR
- Global authentication middleware enforced before routing to controllers

Guard check: a structural audit MUST scan all controllers for mutating verbs lacking authorization enforcement and fail the build.

## SUSPECTED LOCATIONS
- src/platform/api/controllers/TodoController.cs (no `[Authorize]` on Create/Update/Complete)
- src/platform/host/Program.cs (no global auth middleware registered)
