# R2.A OPA Adapter Classification — 2026-04-19

Captured per CLAUDE.md $1c and R2 entry criteria in [`runtime-upgrade-plan.md`](../project-topics/v2b/runtime-upgrade-plan.md) §3.

Closes the implementation loop for **POL-FAIL-CLASS-01** on the OPA adapter side. R2.A.1 added the retry primitive; R2.A.2 added the `PolicyFailureMode` enum + `PolicyEvaluationDeferred` category. This rule wires the OPA unavailable exception through the retry primitive inside `PolicyMiddleware`.

## R-POL-OPA-RETRY-01 — OPA unavailability goes through IRetryExecutor

- **CLASSIFICATION:** runtime / policy / resilience
- **SEVERITY:** S0 (constitutional extension — closes POL-FAIL-CLASS-01's "infinite retry on policy unavailable" + "silent bypass" concerns)
- **DESCRIPTION:**
  - `PolicyMiddleware.ExecuteAsync` MUST wrap the `IPolicyEvaluator.EvaluateAsync` call in `IRetryExecutor.ExecuteAsync`. When the evaluator throws `PolicyEvaluationUnavailableException`, the retry-step outcome is `RetryStepResult<PolicyDecision>.Failure(RuntimeFailureCategory.PolicyEvaluationDeferred, message)`.
  - `RetryEligibility.IsRetryable(PolicyEvaluationDeferred)` returns `true` (set in R2.A.2) so the executor issues bounded attempts (default `MaxAttempts=3`, `InitialDelayMs=200`). Deterministic backoff+jitter preserves replay determinism per R-RETRY-DET-01.
  - On `RetryOutcome.Exhausted` OR `RetryOutcome.PermanentFailure` with category `PolicyEvaluationDeferred`, `PolicyMiddleware` MUST **re-throw** a fresh `PolicyEvaluationUnavailableException` with `reason = "retry_exhausted:<original_reason>"` and `RetryAfterSeconds` preserved from the last caught exception. Rationale: OPA was not reachable — the policy was NOT actually denied, it could not be evaluated. Surfacing as HTTP 503 + `Retry-After` (via the existing `PolicyEvaluationUnavailableExceptionHandler` at the API edge) is more accurate than 403 Forbidden. The POL-FAIL-CLASS-01 invariant "DEFER converts to FAIL_CLOSED" is satisfied via the rejection path — the request never reaches the engine.
  - Policy primacy ($8) is preserved: re-throwing at middleware exhaustion means the API edge still fail-closes; no engine dispatch occurs without a successful `PolicyDecision`.
  - `IRetryExecutor` is an OPTIONAL constructor dependency on `PolicyMiddleware`. When null (legacy test hosts without the executor registered), behaviour falls back to pre-R2.A.2 pass-through — the exception bubbles. Production composition wires the executor via `CoreComposition` (already registered in R2.A.2).
- **NON-GOALS / FORBIDDEN:**
  - **FAIL_OPEN is never applied here.** A successful `CommandResult.Success` MUST NOT result from an OPA unavailability path. POL-FAIL-CLASS-01 permits FAIL_OPEN only when the policy itself declares fail-open-eligibility; R2.A.OPA does not implement that mechanism (separate batch).
  - **WhycePolicyEngine is NOT wrapped in retry.** The constitutional engine is in-process; an exception there is a real defect, not transient unavailability.
  - **The retry loop is bounded.** `MaxAttempts=3` cap honored per R-RETRY-CAP-01. No "infinite retry on unavailable" path exists.
- **CHECK:**
  - Unit test: `PolicyMiddleware_Retries_On_OPA_Unavailable_And_Succeeds_On_Second_Attempt` passes.
  - Unit test: `PolicyMiddleware_Exhausts_Retry_And_Rethrows_PolicyEvaluationUnavailable_With_Retry_Hint_Preserved` passes.
  - Unit test: `PolicyMiddleware_Does_Not_Retry_On_Genuine_OPA_Deny` passes (deny path stays fast — no retry injected on normal allow/deny).
  - Architecture guard: `grep -Pn "catch\s*\(\s*PolicyEvaluationUnavailableException" src/runtime/middleware/policy/PolicyMiddleware.cs` returns ≥ 1.
- **DOC CONTRACT UPDATE:** `PolicyEvaluationUnavailableException` doc comment updated to reflect the R2.A.OPA retry path ("caught inside PolicyMiddleware and wrapped in IRetryExecutor; only surfaces to the API edge after retry exhaustion").
