# R1 Additions — 2026-04-19

Captured per CLAUDE.md $1c during execution of [runtime-upgrade-plan.md](../project-topics/v2b/runtime-upgrade-plan.md) Phase R1.

Two new invariants were added to R1 scope at the user's direction (2026-04-19 approval message) and need to graduate into canonical guards during R1 exit:

1. **DET-RAND-01** — Deterministic randomness seam (`IRandomProvider`) mandatory for all runtime/engine/domain/adapter randomness.
2. **POL-FAIL-CLASS-01** — Policy evaluation failures must classify into exactly one of `FAIL_CLOSED`, `FAIL_OPEN`, or `DEFER`. No silent bypass; no infinite retry on "policy unavailable".

---

## DET-RAND-01 — Deterministic randomness seam

- **CLASSIFICATION:** constitutional / determinism
- **SOURCE:** R1 adjustment 2026-04-19; required to unblock R2 retry jitter, partition-rebalance tiebreaks, circuit-breaker half-open probes, load-shedding victim selection.
- **SEVERITY:** S0 — replay determinism breaks without this; chaos tests become non-reproducible.
- **DESCRIPTION:**
  - Current determinism block list in `constitutional.guard.md` covers `Guid.NewGuid`, `DateTime.UtcNow`, and `Random` instantiation — but there is no approved replacement for legitimate randomness needs (retry jitter, etc.). Without a seam, engineers will either re-introduce `Random` (violating the block list) or hard-code fixed delays (losing backoff benefits).
  - `IRandomProvider` is introduced as the single permitted randomness seam, modeled on `IClock` and `IIdGenerator`. It lives in `src/shared/kernel/domain/IRandomProvider.cs` with seed-driven methods (`NextDouble`, `NextInt`, `NextLong`). The seed MUST derive from deterministic operation coordinates (e.g. `$"{correlationId}:retry:{attempt}"`).
  - In R1 the contract exists but has no production callers. R2 retry/jitter/rebalance paths become the first consumers.
- **PROPOSED_RULE:**
  - Extend `constitutional.guard.md` § Determinism § Block List with: "`Random` / `Random.Shared` / `RandomNumberGenerator.GetBytes` for non-cryptographic purposes remain forbidden; use `IRandomProvider` instead."
  - Extend `constitutional.guard.md` § Determinism § Required Replacements with: "For randomness: `IRandomProvider.NextDouble/NextInt/NextLong` seeded from operation coordinates."
  - Extend `constitutional.guard.md` § Determinism § Exceptions with the `DeterministicRandomProvider` implementation as the single permitted reader of any RNG source.
  - Add a new audit checklist entry in `constitutional.audit.md` § Determinism for `IRandomProvider` usage.
- **RELATED GUARD RULES:** GE-01 deterministic execution, DET-STOPWATCH-OBSERVABILITY-01 (pattern analog)
- **STATUS:** Contract landed in R1 (`src/shared/kernel/domain/IRandomProvider.cs`). Host composition implementation (`DeterministicRandomProvider`) deferred until the first real caller lands in R2.

## POL-FAIL-CLASS-01 — Policy-failure classification invariant

- **CLASSIFICATION:** constitutional / policy
- **SOURCE:** R1 adjustment 2026-04-19; user flagged hidden interaction between policy outage and retry system.
- **SEVERITY:** S0 — a silent bypass under "policy unavailable" undermines WHYCEPOLICY authority (POL-02). An infinite retry loop on a down policy engine is a DoS amplifier.
- **DESCRIPTION:**
  - Spec §6 Policy Enforcement lists "Policy fallback posture rules", "Policy dependency health awareness", and "Policy outage handling discipline" as ABSENT. Spec §14 Failure Handling lists "Recovery-after-policy-outage behavior" as ABSENT.
  - Combining a retry system (R2) with an absent policy-failure taxonomy creates two realistic failure modes:
    1. **Infinite retry:** every policy evaluation returns "unavailable" during an OPA outage; retry logic treats it as transient; loop burns until backoff cap is hit per request × concurrency.
    2. **Silent bypass:** degraded-mode code path skips policy evaluation "to keep the system responsive"; commands mutate state without authorization evidence.
  - Both are S0 by WBSM constitutional rules (POL-02 "No unauthorized domain actions", PB-09 "Policy → chain link required"). Neither can happen once classification is enforced.
- **PROPOSED_RULE:**
  - Every policy evaluation outcome resolves to exactly one of three categories, encoded in a new enum `PolicyFailureMode` under `src/runtime/contracts/`:
    - `FAIL_CLOSED` — reject the command; emit `PolicyEvaluationFailedEvent`; no retry. Default for all production environments.
    - `FAIL_OPEN` — permit the command under an explicit, audited `DegradedPosture.PolicyFailOpen` flag; emit `PolicyBypassedEvent` with reason. Only permitted if WHYCEPOLICY itself declares this policy as fail-open-eligible (breaking-glass operations, emergency read paths). Default is never fail-open.
    - `DEFER` — enqueue the command for bounded retry (attempt cap, backoff); emit `PolicyEvaluationDeferredEvent`. Used when policy engine health is DEGRADED but recoverable.
  - The retry executor (R2, D1) MUST consult `FailureCategory == PolicyDenied` (terminal — no retry) vs a new `FailureCategory == PolicyEvaluationDeferred` (retry with cap). The retry cap MUST be finite.
  - `IPolicyEvaluator` contract MUST return the classification as a first-class field, not embed it in an error string.
  - Audit probe: any code path that catches a policy-unavailable exception and proceeds without emitting one of the three events = S0.
- **PROPOSED_TESTS:**
  - Unit: policy evaluator returns `DEFER` under injected OPA-503; retry executor honors cap; final attempt produces `FAIL_CLOSED` rejection.
  - Integration: kill OPA mid-command; verify `FAIL_CLOSED` event lands in event store; no mutation applied.
  - Chaos (R5): sustained OPA outage under load — no infinite retry, no silent bypass, observable reject rate matches expectation.
- **RELATED GUARD RULES:** POL-02 (no unauthorized domain actions), POL-03 (policy enforcement through runtime), POL-AUDIT-14 (policy evaluation emits event), PB-09 (policy → chain link required)
- **STATUS:** Documented in R1 (no implementation). R2 work must implement the enum and enforce the classification before any retry primitive ships.
