# R2.A.D.2 — OPA Evaluator Refactor (2026-04-19)

Captured per CLAUDE.md $1c. R2.A.D.1 shipped the canonical `ICircuitBreaker` + `DeterministicCircuitBreaker`. R2.A.D.2 refactors `OpaPolicyEvaluator` (the first real consumer) to delegate breaker state to the injected contract, removing the inline state machine. This validates the contract against the proven phase1.5-S5.2.1 pattern.

## R-OPA-BREAKER-DELEGATION-01 — OpaPolicyEvaluator consumes ICircuitBreaker

- **CLASSIFICATION:** runtime / resilience / composition
- **SEVERITY:** S1
- **DESCRIPTION:**
  - `OpaPolicyEvaluator` constructor accepts an injected `ICircuitBreaker` parameter. The evaluator does NOT own the breaker state — it delegates to the contract.
  - Inline breaker state REMOVED: `_breakerLock`, `_consecutiveFailures`, `_openedAt` fields, `IsBreakerOpenInternal`, `RecordSuccess`, `RecordFailure` private methods. Architecture guard grep confirms these tokens are absent post-refactor.
  - The HTTP call inside `EvaluateSingleAsync` is wrapped in `_breaker.ExecuteAsync<PolicyDecision>(...)`. The lambda body preserves the existing transient-failure throw sites (timeout / transport / http_status) — the breaker counts these as failures.
  - `CircuitBreakerOpenException` thrown from `_breaker.ExecuteAsync` is CAUGHT at the evaluator boundary and TRANSLATED to `PolicyEvaluationUnavailableException(reason: "breaker_open", retryAfterSeconds, ...)` preserving the pre-refactor external contract for the R2.A OPA retry path and the API-edge 503 handler.
  - The public `IsBreakerOpen` property is retained with adapted semantics: returns `true` when `_breaker.State != CircuitBreakerState.Closed` (i.e. Open or HalfOpen). This matches the pre-refactor semantics (`_openedAt is not null`) for the `IRuntimeStateAggregator` HC-2 consumer.
- **CHECK:**
  - Architecture test `OpaPolicyEvaluator_Delegates_Breaker_State_To_ICircuitBreaker` scans `src/platform/host/adapters/OpaPolicyEvaluator.cs` and asserts:
    - Contains `ICircuitBreaker` (type reference).
    - Does NOT contain `_consecutiveFailures`, `_openedAt`, `_breakerLock` (inline state removed).
    - Contains the translation path `catch\s*\(\s*CircuitBreakerOpenException` (boundary translation preserved).
  - Existing `PolicyEvaluationRetryPatternTests` (6 tests) continue to pass — they use a scripted fake evaluator that still throws `PolicyEvaluationUnavailableException` directly, so the retry-around-evaluator pattern is regression-guarded.
  - Host composition registers a named `ICircuitBreaker` for OPA (`"opa-policy-evaluator"`) with options derived from `OpaOptions.BreakerThreshold` / `BreakerWindowSeconds` — preserving the existing configuration surface.

## R-OPA-BREAKER-BOUNDARY-01 — CircuitBreakerOpenException does NOT leak from OpaPolicyEvaluator

- **CLASSIFICATION:** runtime / resilience / boundary
- **SEVERITY:** S1
- **DESCRIPTION:**
  - `OpaPolicyEvaluator.EvaluateAsync` MUST NOT propagate `CircuitBreakerOpenException` to callers. The typed exception used by the OPA adapter is `PolicyEvaluationUnavailableException` — existing callers (PolicyMiddleware retry wrapper, API-edge exception handler) depend on that contract.
  - The translation happens at the adapter boundary inside `EvaluateSingleAsync`. `CircuitBreakerOpenException.RetryAfterSeconds` is copied onto the new `PolicyEvaluationUnavailableException.RetryAfterSeconds` so Retry-After guidance flows through.
- **CHECK:**
  - Any grep of `throw.*CircuitBreakerOpenException` inside `OpaPolicyEvaluator.cs` returns zero hits (breaker instance throws; evaluator catches + translates; never throws it).
  - `PolicyEvaluationRetryPatternTests.Exhausts_Retry_And_Rethrows_PolicyEvaluationUnavailable_With_Retry_Hint_Preserved` — semantics flow through unchanged under the refactor.
