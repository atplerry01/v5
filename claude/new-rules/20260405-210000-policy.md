SOURCE: T0U identity/policy implementation audit sweep (2026-04-05)
DESCRIPTION: Policy evaluation in T0U WhycePolicyEngine does not emit a PolicyEvaluatedEvent or PolicyDeniedEvent for audit trail purposes
PROPOSED_RULE: Runtime pipeline MUST emit a policy audit event (PolicyEvaluatedEvent or PolicyDeniedEvent) after T0U policy evaluation completes, regardless of allow/deny outcome. This event must include DecisionHash, IdentityId, PolicyName, and IsAllowed.
SEVERITY: S1
---

PolicyMiddleware calls WhycePolicyEngine and injects the decision into CommandContext, but no domain event is persisted for the policy evaluation itself. The DecisionHash is propagated to the chain anchor via CommandContext, but the decision is not independently auditable as an event. This should be addressed when the runtime event emission pipeline is extended to support pre-execution audit events.
