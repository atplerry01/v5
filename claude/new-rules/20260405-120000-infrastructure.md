## FINDING 1: Health Check Adapter Placement Rule

**PROPOSED_RULE:** S-HEALTH-01 — All infrastructure health checks MUST be adapter implementations in `infrastructure/health/`. Health check contracts (IHealthCheck, HealthCheckResult) belong in `shared/contracts/infrastructure/health/`. Health check composition (HealthAggregator) belongs in `infrastructure/` and is accessed only through platform API endpoints.
**SEVERITY:** S2 (MEDIUM)
**DESCRIPTION:** New canonical placement rule for health check infrastructure. Prevents health checks from leaking into domain or runtime layers.

## FINDING 2: Observability Middleware Placement Rule

**PROPOSED_RULE:** S-OBSERVABILITY-01 — Observability middleware (metrics, logging, tracing) MUST be isolated in `runtime/observability/` and registered in the runtime middleware pipeline. Middleware MUST NOT contain business logic or domain references.
**SEVERITY:** S2 (MEDIUM)
**DESCRIPTION:** New canonical placement rule for observability cross-cutting concerns.
