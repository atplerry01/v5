# ACT-FABRIC-ROUNDTRIP-TEST-01 — Every registered event must have a real-fabric round-trip test

## DESCRIPTION

The phase-1-gate sweep found that `WorkflowExecutionResumedEvent` was emitted by the aggregate, handled in `LoadFromHistory()`, and exercised by `WorkflowReplayResumeTests.cs` — and yet *was not registered with the event fabric at all*. The integration test passed because it short-circuits the fabric with in-memory doubles. In production, a workflow resume would fail at deserialization.

This is the most dangerous class of activation drift: **tests that green-light a path that does not exist outside the test harness**. Static activation-registry checks are necessary but not sufficient — the registry can be passed without the fabric ever being exercised end-to-end for a given event.

## PROPOSED_RULE

1. Every domain event whose schema is registered with `IEventSchemaRegistry` (or equivalent) MUST have at least one integration test that:
   - Constructs the event
   - Persists it via the real `IEventFabric` (not a double)
   - Reloads it from the real event store
   - Asserts payload integrity round-trip
2. Tests that bypass the real fabric (by injecting in-memory event stores or stub schema registries) MUST be tagged with a category attribute (e.g. `[Trait("Fabric", "Bypass")]`).
3. A CI gate enumerates all registered events and asserts each has at least one non-bypass test exercising it. Any event present in the registry but absent from the test set fails the build.

## ENFORCEMENT

- Test-discovery scan in CI: enumerate the test set, partition into bypass vs real-fabric, build coverage map per event type, fail on uncovered registrations.
- Add a unit test in `tests/unit/architecture/` that asserts every event class has a matching `[Fact]` somewhere in `tests/integration/` referencing it through the real fabric.

## SEVERITY

S0 — system-breaking. The whole point of the activation registry is to make sure the fabric knows about every event. A test harness that bypasses the fabric while claiming to validate the event makes the registry's guarantees meaningless. This rule exists specifically to prevent the S0.1 finding from this sweep from ever recurring.
