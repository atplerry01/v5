# R2.A.D.3c — Postgres Pool Circuit Breaker (2026-04-19)

Captured per CLAUDE.md $1c. This is the fifth and final dependency-level breaker in the R2.A.D story (OPA → Chain → Kafka → Redis → Postgres). User approved Option A on 2026-04-19: shared `"postgres-pool"` breaker at the connection-acquire seam.

## R-POSTGRES-POOL-BREAKER-01 — Shared "postgres-pool" breaker wraps all pooled acquisitions

- **CLASSIFICATION:** runtime / resilience / postgres
- **SEVERITY:** S1
- **DESCRIPTION:**
  - One `ICircuitBreaker` instance named `"postgres-pool"` registered in host composition alongside the three `NpgsqlDataSource` wrappers. Registered as plain `ICircuitBreaker` so `CircuitBreakerRegistry` picks it up automatically.
  - Breaker is applied at the canonical acquisition seam: `EventStoreDataSource.OpenAsync(ct)`, `ChainDataSource.OpenAsync(ct)`, and `ProjectionsDataSource.OpenAsync(ct)` each wrap `Inner.OpenInstrumentedAsync(PoolName, ct)` inside `_breaker.ExecuteAsync(...)` when a breaker is injected.
  - A single shared breaker (not per-pool) is chosen because the three declared pools typically point at the same Postgres instance in deployments — a Postgres outage affects all three uniformly and should produce one coherent health signal, not three competing ones. The pool-level `TotalFailures` counters (HC-6) remain per-pool for diagnostic attribution.
  - The R2.A.D.4 aggregator whitelist adds `"postgres-pool"` → `"postgres_pool_breaker_open"`.
  - **Scope of breaker-counted failures**: every exception thrown by `Inner.OpenInstrumentedAsync` (timeout, transport, pool exhaustion, auth) counts toward the breaker threshold. `CircuitBreakerOpenException` itself is NOT counted (breaker aborts before the operation runs).
- **CHECK:**
  - `sp.GetService<ICircuitBreakerRegistry>()?.Get("postgres-pool")` returns the breaker.
  - All 14 host-adapter call sites that previously called `_dataSource.Inner.OpenInstrumentedAsync(PoolName, ct)` now call `_dataSource.OpenAsync(ct)`.
  - `PostgresProjectionStore` (projections assembly) accepts optional `ICircuitBreaker?` ctor parameter and wraps its local `OpenInstrumentedAsync` similarly.
  - The canonical `PostgresPoolMetrics.OpenInstrumentedAsync` instrumentation (acquisition / failure counters, HC-6 PoolStateBucket, StateChange callback) is unchanged — the breaker wraps it, it doesn't replace it.

## R-POSTGRES-POOL-BREAKER-OPEN-SEMANTICS-01 — Per-adapter Open-state handling

- **CLASSIFICATION:** runtime / resilience / postgres / boundary
- **SEVERITY:** S1
- **DESCRIPTION:**
  - `CircuitBreakerOpenException` is caught by each adapter's existing outer try/catch and translated per the posture table:
    | Adapter | Open-state action | Rationale |
    |---|---|---|
    | `PostgresOutboxAdapter` (enqueue) | Re-throw (transaction rolls back) | Outbox enqueue is inside the command transaction — failure must propagate so `CommandResult.Failure(DependencyUnavailable)` is returned and the caller retries. |
    | `KafkaOutboxPublisher` (background drain) | Log + skip this iteration | Background publisher — skip-not-fail; the next poll cycle tries again. Unpublished rows stay in the outbox. |
    | `OutboxDepthSampler` (background sampler) | Log + skip this iteration | Same as publisher — best-effort metric sampling. |
    | `PostgresIdempotencyStoreAdapter` | Re-throw as `DependencyUnavailable` | Idempotency check must be authoritative. A skipped check can produce duplicate events — fail the command. |
    | `PostgresSequenceStoreAdapter` | Re-throw as `DependencyUnavailable` | Sequence allocation is a hard dependency — cannot proceed without it. |
    | `PostgresEventStoreAdapter` | Re-throw as `DependencyUnavailable` | Event store is the authoritative write target — fail the command. |
    | `WhyceChainPostgresAdapter` | Re-throw as `DependencyUnavailable` | Chain anchor in the command critical path — fail the command. |
    | `PostgresAdvisoryLeaseProvider` (TryAcquireAsync) | Return null (busy signal) | Matches existing "lease unavailable" contract — callers fall back to local path or retry. |
    | `PostgresDeadLetterStore` (StoreAsync) | Log + swallow | DLQ store is best-effort; losing the mirror must not block the Kafka DLQ publish. |
    | `PostgresDeadLetterStore` (read paths) | Re-throw | Admin/operator read paths — callers need to know the store is unavailable. |
    | `PostgresProjectionWriter` | Re-throw | Kafka consumer worker catches and routes to DLQ per existing poison-message handling. |
    | `PostgresProjectionStore` (projections assembly) | Re-throw | Same — projection handler's error path already exists. |
  - Each adapter's catch branch for `CircuitBreakerOpenException` is SEPARATE from the generic `catch (Exception)` branch so the breaker-open case can be logged and metered distinctly from organic DB errors.
- **CHECK:**
  - Grep: each adapter file contains exactly one `catch (CircuitBreakerOpenException` branch (or an explicit re-throw path). Architecture guard is a simple find.
  - Outbox enqueue: `CircuitBreakerOpenException` is allowed to propagate (do NOT catch in enqueue — the transaction rollback depends on it).
  - DLQ store: `CircuitBreakerOpenException` caught only in `StoreAsync` — read paths propagate.

## R-POSTGRES-POOL-BREAKER-DATASOURCE-API-01 — DataSource wrappers expose OpenAsync(ct) as the canonical acquire

- **CLASSIFICATION:** runtime / resilience / api
- **SEVERITY:** S2
- **DESCRIPTION:**
  - `EventStoreDataSource`, `ChainDataSource`, `ProjectionsDataSource` each expose an `OpenAsync(CancellationToken)` method. This is the canonical connection-acquire API for all host adapters after R2.A.D.3c.
  - The pre-R2.A.D.3c pattern `_dataSource.Inner.OpenInstrumentedAsync(PoolName, ct)` is DEPRECATED for new adapters. Existing references in README.md and historical audit outputs are HISTORICAL BASELINE.
  - The `Inner` property remains public because the projections assembly (which cannot reference host-adapters) consumes the raw `NpgsqlDataSource` directly and applies its own breaker wrap.
- **CHECK:**
  - Grep for `\.Inner\.OpenInstrumentedAsync\(` in `src/platform/host/adapters/**` returns zero hits after this refactor (the seam remains for the `PostgresProjectionStore` in the projections assembly, which receives the inner data source directly).
  - Each DataSource wrapper has a public `OpenAsync(CancellationToken ct = default)` method.
  - The wrapper's ctor accepts optional `ICircuitBreaker? poolBreaker` parameter (trailing, default null) so existing tests that construct the wrapper without a breaker continue to compile.

## Out of scope for R2.A.D.3c

- **Per-pool breakers** — intentionally NOT implemented. User-approved design is a single shared breaker. If one pool's failure profile diverges enough to need its own breaker, it will ship as a separate rule.
- **Pool-level failure attribution** — HC-6 already provides per-pool `TotalFailures` / `RecentFailures`. No new attribution plumbing needed.
- **.retry / .deadletter tier Postgres tables** — deferred to R3.

## Promotion path

Rules promoted to `runtime.guard.md` "R2.A.D Backpressure & Circuit Protection" section. Audit entries added to `runtime.audit.md` Section 14. This closes R2.A.D — 5 of 5 dependency breakers shipped.
