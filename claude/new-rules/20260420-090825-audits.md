## Rationale

1. Target canonical tree lists retention only under `document/governance/`.
   No retention entry under `shared/` or under `media/` or `streaming/`.
2. Retention targets differ materially by context: document retention
   targets a document/file/record; media retention would target an asset/
   rendition; streaming retention would target an archive. Each target set
   has its own governance/legal model.
3. Shared context is reserved for truly cross-context graph concepts
   (relationship, localization, evidence). Retention is a compliance
   concern whose rules differ by content kind — not a cross-context truth.

## Applied

`document/governance/retention` is LIVE (moved in CS.2 from
`document/lifecycle/retention`, 15 files rewrote namespace from
`ContentSystem.Document.Lifecycle.Retention` to
`ContentSystem.Document.Governance.Retention`). No shared scaffold for
retention.
