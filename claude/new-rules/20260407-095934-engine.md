# NEW RULE — EVENT VERSION MUST INCREMENT

CLASSIFICATION: event-sourcing / versioning / engine
SOURCE: E2E lifecycle validation (todo create + update), 2026-04-07
SEVERITY: S1

## DESCRIPTION
All events for a given aggregate MUST have strictly increasing version numbers.

Observed violation:
- `TodoCreatedEvent` → version 0
- `TodoUpdatedEvent` → version 0 (expected 1)

Both events persisted under the same `aggregate_id` with identical version, breaking event-sourcing invariants and optimistic concurrency.

## PROPOSED_RULE
Version MUST be derived from aggregate state at append time:

    newVersion = currentVersion + 1

The persistence layer MUST NOT override or reset the engine-computed version. Engines MUST load prior aggregate state before applying mutating commands.

## SUSPECTED LOCATIONS
- src/engines/T2E/operational/todo/TodoEngine.cs (update path may not load prior version)
- Postgres event store adapter (verify it persists supplied version, not a default)
