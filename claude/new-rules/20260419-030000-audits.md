## TESTS-UNIT-RUNTIME-DISABLED-01 — Pre-existing disabled test suite

- **CLASSIFICATION:** audits / test-coverage
- **SEVERITY:** S2 (historical — closed in this capture)
- **DESCRIPTION:**
  - `tests/unit/Whycespace.Tests.Unit.csproj` carried a blanket `<Compile Remove="runtime\**\*.cs" />` exclusion, justified by a comment stating the files "transitively depend on Whycespace.Shared or Whycespace.Engines, which are intentionally not referenced."
  - Re-inspection during R1 Batch 5: **five** of the six pre-existing runtime test files import ONLY `Whycespace.Shared.Contracts.*` — already transitively available via `Whycespace.Tests.Shared` → `Whycespace.Shared`. They compile unchanged under the existing reference graph. The exclusion was overzealous.
  - Affected tests that had been silently disabled:
    - `tests/unit/runtime/CommandContextReplayResetTests.cs` — 9 tests pinning `CommandContext` write-once + replay reset semantics (central to phase1.6-S1.4 correctness).
    - `tests/unit/runtime/ExecutionLockProviderTests.cs` — execution lock concurrency.
    - `tests/unit/runtime/RuntimeEnforcementGateTests.cs` — maintenance + degraded-mode enforcement gate.
    - `tests/unit/runtime/RedisHealthCheckTests.cs` — HC-9 Redis health posture.
    - `tests/unit/runtime/PostgresPoolHealthEvaluatorTests.cs` — Postgres pool saturation detection.
    - `tests/unit/runtime/RuntimeDegradedModeTests.cs` — HC-7 degraded-mode classification.
  - Net effect: ~40 unit tests covering runtime invariants (including the replay-determinism seam) were present on disk but not part of CI. Any drift in these invariants would have been silent.
- **CLOSURE (R1 Batch 5, 2026-04-19):**
  - Removed `<Compile Remove="runtime\**\*.cs" />` from `tests/unit/Whycespace.Tests.Unit.csproj`.
  - Added `<ProjectReference Include="..\..\src\runtime\Whycespace.Runtime.csproj" />` so R1 Batch 5's new `IdempotencyConcurrencyStressTests` (which imports `Whycespace.Runtime.Middleware.PostPolicy.IdempotencyMiddleware`) compiles.
  - Replaced the `<Compile Remove>` comment with an R1 rationale and dated closure marker.
  - Verified: 46/46 runtime unit tests pass (40 pre-existing + 6 R1-era additions).
