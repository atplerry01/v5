# E-LIFECYCLE-FACTORY-CALL-SITE-01 — T1M engines must not invoke aggregate command methods

## DESCRIPTION

T1M engines are stateless event producers. Their contract is: receive command + read model, produce events. They must not mutate aggregates directly — that responsibility belongs to the runtime persist pipeline applying the events the engine produced.

`WorkflowExecutionReplayService.ResumeAsync()` at `src/engines/T1M/lifecycle/WorkflowExecutionReplayService.cs:111` violates this by calling `aggregate.Resume()` directly. The reason it took the shortcut is informative: `WorkflowLifecycleEventFactory` is missing a `Resumed()` method, so there was no factory seam to use. The fix and the rule are entangled — you cannot enforce the rule without ensuring every lifecycle transition has a corresponding factory method.

## PROPOSED_RULE

1. T1M engines (and any future stateless engine tier) MUST NOT invoke command methods on domain aggregates. Allowed aggregate interactions are limited to:
   - Reading state via query methods / properties
   - Calling `LoadFromHistory()` for replay reconstruction
2. All state transitions a T1M engine wishes to express must be produced by calling a lifecycle event factory method that returns the corresponding event record.
3. For every aggregate command method `Aggregate.X(...)`, there MUST exist a corresponding `LifecycleFactory.X(...)` returning the event that command would have raised. Adding a new aggregate command without adding the factory method is a guard violation.

## ENFORCEMENT

- Static check: in `src/engines/**`, flag any method call whose receiver type derives from `AggregateRoot` and whose target is not in the read-only allowlist.
- Pairing check: for every public command method on every aggregate, assert the existence of a matching factory method by name. Add to `WbsmArchitectureTests`.

## SEVERITY

S1 — architectural, blocks merge. Direct aggregate mutation from engines violates layer purity ($7) and produces a class of bugs where the engine's view of aggregate state diverges from the persisted event stream.
