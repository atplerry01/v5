## NEW RULE FINDING

**CLASSIFICATION**: runtime  
**SOURCE**: Post-execution audit sweep — Todo E2E Pipeline Alignment  
**DESCRIPTION**: PolicyMiddleware used `context = context with { PolicyDecisionHash = ... }` which creates a new record instance. Because the RuntimeCommandDispatcher captures context in closures, the mutated copy was invisible to downstream pipeline stages (ExecuteEngineAsync). This caused the ChainAnchor to receive an empty DecisionHash, violating GE-03 (WHYCECHAIN ANCHORING).  
**PROPOSED_RULE**: CommandContext.PolicyDecisionHash MUST be mutable (set, not init) so that middleware can mutate the captured context instance in-place. Alternatively, the middleware pipeline must propagate context changes through the pipeline delegate chain rather than relying on closure capture of a single variable.  
**SEVERITY**: S1 — HIGH (chain integrity compromised; DecisionHash missing from audit trail)  
**RESOLUTION**: Changed `PolicyDecisionHash { get; init; }` → `{ get; set; }` and PolicyMiddleware from `context = context with { ... }` → `context.PolicyDecisionHash = ...`.
