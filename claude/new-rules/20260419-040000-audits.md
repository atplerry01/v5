## AUDIT-PROBE-COMMENT-STRIP-01 — Audit probes must strip comments before matching

- **CLASSIFICATION:** audits / probe-hygiene
- **SEVERITY:** S3
- **DESCRIPTION:**
  - Section 13 of `runtime.audit.md` (R1 Replay Determinism Certification) provides probes like:
    ```
    grep -RnP "\bDateTime(\.UtcNow|\.Now)\b|\bDateTimeOffset(\.UtcNow|\.Now)\b" src/runtime src/engines src/domain src/platform/host/adapters | grep -v "SystemClock"
    ```
  - These probes are LINE-LEVEL with no comment-stripping, so they match:
    - Comment blocks describing what the file *must not* do (`// MUST NOT call DateTime.UtcNow`)
    - Documentation XML comments (`/// never <c>DateTime.UtcNow</c>`)
    - Source code in strings / attributes
  - The R1 audit sweep (2026-04-19) produced 4 "CONDITIONAL" findings that on manual inspection turned out to all be doc comments — zero actual drift. False-positive rate 4/4 on the DateTime probe.
  - By contrast, `WbsmArchitectureTests.ScanCode` (`tests/unit/architecture/WbsmArchitectureTests.cs`) uses a `StripCommentAndString` helper that removes `//` comments and simple `"..."` string literals before regex matching. This approach already lives in the codebase; the audit-doc probes should use the same discipline.
- **PROPOSED_RULE:**
  - Every grep-based probe in `runtime.audit.md` (and peer audit files) that scans source files for forbidden tokens MUST either:
    - (a) pipe through a comment-stripping filter (e.g. `sed 's|//.*$||'` then `grep`), or
    - (b) reference an architecture test that does the stripping (e.g. "Probe: `WbsmArchitectureTests.<TestName>` passes").
  - Add to `runtime.audit.md` § Output Format: "Grep-based probes that match comment spans or XML doc references MUST be refined before reporting FAIL. Prefer architecture-test references over raw grep where a test exists."
  - Raise to S2 once any new audit probe is written without comment-stripping.
- **ACTION IN R2:**
  - No code action required today. This is a documentation / probe-hygiene rule.
  - When Section 13 is next edited (next R1 sweep or R2 entry re-check), update the `DateTime` / `Guid.NewGuid` / `Random` probes to pipe through comment-stripping OR replace with architecture-test references.
- **RELATED GUARD RULES:**
  - Architecture test `IdGenerator_Generate_seeds_contain_no_clock_or_random_entropy` already uses the correct discipline — reference implementation.
  - Test Architecture rules (isolation + determinism) in `runtime.guard.md`.
