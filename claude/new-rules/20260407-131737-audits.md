# FINDING: Determinism block list does not cover platform host adapters

## DESCRIPTION

The Phase 2 hardening sweep ran a baseline violation inventory and found that
the canonical determinism rules (behavioral.guard.md rule 16, GE-01 across
domain/engine/runtime guards, and the runtime C# `DeterminismGuard.cs`) only
forbid `Guid.NewGuid()` / `DateTime.UtcNow` / `DateTimeOffset.UtcNow` in
`src/{domain,engines,runtime}/`. Two real violations were found in
`src/platform/host/adapters/`:

- `PostgresEventStoreAdapter.cs:82` — `Guid.NewGuid()` used as the row id
  for the events table INSERT. This breaks deterministic replay: the same
  domain event appended twice (replay scenarios, disaster recovery, multi-region
  rebuilds) produces different physical row ids, which complicates
  idempotency, deduplication, and chain anchoring.

- `GenericKafkaProjectionConsumerWorker.cs:94` — `DateTimeOffset.UtcNow` used
  to stamp the `EventEnvelope.Timestamp` for projection envelopes consumed
  from Kafka. This couples projection state to wall-clock time at consume
  moment, which is non-deterministic on rebuild.

Both files are infrastructure adapters at the composition root. They are
*technically* outside the canonical block list, but they are the primary
persistence and event-fabric boundaries — exactly the surfaces where
non-determinism is most damaging to event sourcing replay guarantees.

`IClock` and `IIdGenerator` already exist in
`src/shared/kernel/domain/` and are already DI-registered as singletons in
`Program.cs:57` and `Program.cs:60` (`SystemClock` and
`DeterministicIdGenerator` respectively). The seams exist; the adapters
simply do not consume them.

## PROPOSED RULE

Extend the canonical determinism block list to cover `src/platform/host/adapters/**`:

> Within `src/platform/host/adapters/**`, the following are FORBIDDEN:
>
> - `Guid.NewGuid()` — must use injected `IIdGenerator.Generate(seed)` with
>   a deterministic seed derived from the aggregate id, version, or stream
>   coordinate.
> - `DateTime.Now`, `DateTime.UtcNow`, `DateTimeOffset.Now`,
>   `DateTimeOffset.UtcNow` — must use injected `IClock.UtcNow`.
>
> EXCEPTIONS:
>
> - The `IClock` implementation itself (`SystemClock` in `Program.cs`) is the
>   single permitted reader of `DateTimeOffset.UtcNow`. It is the boundary
>   between deterministic application code and the system clock.
> - SQL `NOW()` / `CURRENT_TIMESTAMP` inside SQL statements is permitted for
>   audit columns the application does not read back into deterministic logic.

This finding should be promoted into a new canonical guard file
`claude/guards/determinism.guard.md` that consolidates the determinism rules
already scattered across behavioral / domain / engine / runtime guards AND
extends them to the platform-adapter surface.

## FIXES APPLIED IN THIS EXECUTION

1. `PostgresEventStoreAdapter` — constructor now takes `IIdGenerator`; row id
   is derived from `$"{aggregateId}:{version}"`.
2. `GenericKafkaProjectionConsumerWorker` — constructor now takes `IClock`;
   envelope timestamp uses `_clock.UtcNow`.
3. `Program.cs` and `TodoBootstrap.cs` DI wiring updated to inject the new
   dependencies from the existing singleton registrations.
4. `claude/guards/determinism.guard.md` created with the consolidated rule
   set above.
