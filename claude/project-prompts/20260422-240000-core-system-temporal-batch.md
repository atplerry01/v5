Optional batch description:
Canonical temporal primitives — immutable, UTC-normalized representations of points, bounded
ranges, half-open windows, and policy-activation periods. Used universally across all systems
to eliminate ambiguous time representations and enforce timezone-safe, monotonic comparisons.

# OBJECTIVE

Implement and maintain the `temporal` domain group inside `core-system` as pure semantic
primitives: no aggregates with lifecycle, no state transitions, no behavior beyond validation
and comparison. All temporal values must be UTC-normalized at construction, support
deterministic comparison, and be serialization-safe.

Reference: `/claude/project-topics/v3/core-system.md` — sections 1–8, 11–22.

# CONSTRAINTS

1. Domain layer ZERO external dependencies — no infrastructure, no persistence, no messaging.
2. All `DateTimeOffset` values must be normalized to UTC on construction (`ToUniversalTime()`).
3. No `DateTime.Now`, `DateTime.UtcNow`, `DateTimeOffset.UtcNow` inside domain layer.
4. All types must be immutable: `sealed record` or `readonly record struct`.
5. No aggregates with lifecycle, no commands, no events defined here.
6. No domain services, no specifications with behavior, no side effects.
7. core-system may not depend on any other classification.
8. No policy evaluation, no access rules, no orchestration semantics.

# EXECUTION STEPS

## E1 — Domain Primitives

### time-point
- `TimePoint` — wraps `DateTimeOffset`; normalizes to UTC; rejects default; supports
  `IsBefore`, `IsAfter`, `IsSameInstant`, comparison operators.

### time-range
- `TimeRange` — closed interval [Start, End]; start inclusive, end inclusive; UTC-normalized;
  `Duration`, `Midpoint`, `Contains`, `Overlaps`, `Precedes`, `Follows`, `Gap`.

### time-window
- `TimeWindow` — half-open interval [Start, End); open-ended if End is null; UTC-normalized;
  `IsOpen`, `Duration`, `Contains`, `Overlaps`, `ToTimeRange` (when closed).

### effective-period
- `EffectivePeriod` — unbounded-capable policy activation window (both boundaries optional);
  `Always` static, `IsActive`, `HasStarted`, `HasExpired`, `Overlaps`.

## E2–E5 — N/A
core-system has no commands, no queries, no engine handlers, no policy model.

## E6 — N/A
core-system defines no messaging constructs.

## E7 — N/A
core-system primitives are used directly in projections — no read model layer.

## E8 — N/A
core-system is not exposed as an API surface directly — it is consumed as types.

## E10 — Observability
Temporal types must support trace correlation (included in observability payloads as typed
fields). No observability signals generated by the primitives themselves.

## E12 — Validation
- Equality and comparison tests
- UTC normalization tests (non-UTC input → stored as UTC)
- Immutability tests (no public setters)
- Boundary condition tests (zero-duration, open-ended, etc.)
- Serialization round-trip tests
- Overlap / containment / gap computation tests
