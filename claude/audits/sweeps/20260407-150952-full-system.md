## S0 — CRITICAL (5)

### 1. Workflow events bypass persistence pipeline
- **File:** [src/runtime/dispatcher/RuntimeCommandDispatcher.cs:117](src/runtime/dispatcher/RuntimeCommandDispatcher.cs#L117), [src/runtime/dispatcher/RuntimeCommandDispatcher.cs:191](src/runtime/dispatcher/RuntimeCommandDispatcher.cs#L191)
- **Rule:** R-WORKFLOW-PIPE-01 / runtime.guard.md rule 7 (sole persist/publish/anchor)
- **Issue:** `eventsRequirePersistence: false` returned for workflow events. EventFabric never persists, never anchors, never publishes. Execution hash breaks.
- **Fix:** Set `eventsRequirePersistence: true` when `result.EmittedEvents.Count > 0` in both `ExecuteWorkflowAsync` and `ResumeWorkflowAsync`.

### 2. Missing PolicyEvaluatedEvent / PolicyDeniedEvent emission
- **File:** [src/runtime/middleware/policy/PolicyMiddleware.cs:37-102](src/runtime/middleware/policy/PolicyMiddleware.cs#L37-L102)
- **Rule:** POL-AUDIT-01
- **Issue:** Policy decision recorded only in execution context hash; no domain event for independent audit.
- **Fix:** Emit `PolicyEvaluatedEvent` (allow) or `PolicyDeniedEvent` (deny) carrying `{DecisionHash, IdentityId, PolicyName, IsAllowed}` after each evaluation branch.

### 3. EventReplayService uses placeholder policy hash "replay"
- **File:** [src/runtime/event-fabric/EventReplayService.cs:58](src/runtime/event-fabric/EventReplayService.cs#L58)
- **Rule:** CHECK-05.2 / chain immutability
- **Issue:** Hardcoded `PolicyHash = "replay"` breaks WhyceChain traceability for replayed events.
- **Fix:** Preserve the original `PolicyHash` from the persisted envelope, or emit an audit event documenting absence.

### 4. Non-deterministic timestamp in Kafka projection consumer
- **File:** [src/platform/host/adapters/GenericKafkaProjectionConsumerWorker.cs:98](src/platform/host/adapters/GenericKafkaProjectionConsumerWorker.cs#L98)
- **Rule:** PLAT-DET-01
- **Issue:** `Timestamp = _clock.UtcNow` stamps consume time, not event-envelope time. Breaks causality.
- **Fix:** Hydrate Timestamp from the event envelope (or message header), not consume-time clock read.

### 5. T1M step re-entry missing architectural-decision comment
