## S0 — Halt-class findings

### S0.1 — `WorkflowExecutionResumedEvent` not registered with event fabric
- **Audit:** activation
- **Files:** `src/platform/host/composition/orchestration/workflow/WorkflowExecutionBootstrap.cs`, `src/shared/contracts/events/orchestration-system/workflow/WorkflowExecutionEventSchemas.cs`
- **Symptom:** `WorkflowExecutionAggregate.Resume()` emits `WorkflowExecutionResumedEvent`; the aggregate's `LoadFromHistory()` switch handles it; `WorkflowReplayResumeTests.cs` exercises the codepath. But the bootstrap module never calls `schema.Register("WorkflowExecutionResumedEvent", ...)`, never registers a payload mapper, and the projection handler is not wired for it.
- **Runtime impact:** Any workflow resume that round-trips through the fabric will fail at deserialization. The integration test only passes because it short-circuits the fabric with in-memory doubles.
- **Why this is S0 not S1:** A resume path is silently broken in production while passing in tests — that is exactly the class of failure $1a/$11 exist to catch. It is also a *test trust* failure: the test green-lights a path that does not exist outside the test harness.
- **Remediation:**
  1. Add `WorkflowExecutionResumedEventSchema` record to `WorkflowExecutionEventSchemas.cs`
  2. Add `schema.Register(...)` and `schema.RegisterPayloadMapper(...)` calls in `WorkflowExecutionBootstrap.RegisterSchema()`
  3. Wire the projection handler in `WorkflowExecutionBootstrap.RegisterProjections()`
  4. **Add an integration test that round-trips Resume through the real fabric** so this class of activation drift cannot recur

## S1 — Architectural blockers

### S1.1 — Non-deterministic ID seeds (3 sites, one root cause)
- **Audit:** determinism
- **Files & lines:**
  - `src/platform/api/controllers/TodoController.cs:45` — aggregate ID seed includes `_clock.UtcNow.Ticks`
  - `src/runtime/dispatcher/SystemIntentDispatcher.cs:32-35` — correlation/causation/command IDs all seeded with ticks
  - `src/systems/midstream/wss/WorkflowDispatcher.cs` — workflow ID seed includes ticks
- **Pattern:** All three sites correctly use the permitted seam `IIdGenerator.Generate(seed)` — but the seed string itself embeds wall-clock ticks. The generator is deterministic; the seeds are not. Same logical command produces different IDs on replay.
- **Guard violated:** $9 (determinism), DET-SEED-01
- **Remediation:** Seeds must be derived only from stable command coordinates (aggregate id, command type, version, causation id). Never include time, never include `Guid.NewGuid()`. See new-rule candidate `DET-SEED-DERIVATION-01` below.

### S1.2 — Engine layer mutates aggregate directly
- **Audit:** engine
- **File:** `src/engines/T1M/lifecycle/WorkflowExecutionReplayService.cs:111`
- **Violation:** `ResumeAsync()` calls `aggregate.Resume()` directly. T1M engines are stateless event producers — they must construct events via the lifecycle factory, not invoke aggregate methods.
- **Contributing cause:** `WorkflowLifecycleEventFactory` is missing a `Resumed()` method, which is *why* the engine took the shortcut. Both must be fixed together.
- **Guard violated:** E-LIFECYCLE-FACTORY-01, $7 (layer purity)
- **Remediation:** Add `WorkflowLifecycleEventFactory.Resumed(...)` returning `WorkflowExecutionResumedEvent`; replace the direct call site with the factory invocation.

### S1.3 — Event store schema drift (DRIFT-3)
- **Audit:** runtime
- **Files:** `src/runtime/event-fabric/EventFabric.cs:99-114`, `infrastructure/data/postgres/event-store/migrations/*`
- **Violation:** `EventEnvelope` populates `execution_hash`, `correlation_id`, `causation_id`, `policy_decision_hash`, `policy_version` on every persist call, but the `events` table has no columns to receive them. Audit metadata is being silently dropped (or worse, causing the adapter to write to non-existent fields).
- **Cross-reference:** Already noted in `claude/new-rules/_archives/20260408-000000-phase1-gate-blockers.md`.
- **Guard violated:** $11 (audit emission), R-EVENT-AUDIT-COLS-01
- **Remediation:** Migration extending `events` table with the five audit columns + index on `correlation_id`. Must land before Phase 2 integration tests can witness chain integrity.

### S1.4 — `CommandContext` policy fields are write-once-locked *(CONTESTED)*
- **Audit:** policy
- **File:** `src/shared/contracts/runtime/CommandContext.cs:85-96`
- **Subagent finding:** `PolicyDecisionHash`, `PolicyDecisionAllowed`, `PolicyVersion` throw `InvalidOperationException` on second assignment. Subagent argues these are mutable runtime state, not immutable domain values, and the lock blocks legitimate replay/recovery/test paths.
- **Consolidator counter-argument:** Write-once on a policy decision hash is a *defensible* design — it prevents a downstream stage from silently overwriting the upstream policy verdict. That is a real attack/bug surface. The subagent may have over-indexed on test ergonomics over invariant strength.
- **Status:** **NEEDS HUMAN JUDGMENT** before remediation. Do not auto-relax. Possible middle ground: keep the lock, but expose a separate `Reset()` available only to a designated replay seam, gated by an interface marker.

### S1.5 — `OutboxPublisher` MAX_RETRY hardcoded to 5
- **Audit:** kafka
- **File:** `src/platform/host/adapters/PostgresOutboxAdapter.cs` (and/or related publisher)
- **Violation:** No configuration lever for retry budget. Operability/incident-response gap rather than correctness bug, but flagged S1 because under sustained downstream outage there is no operator escape hatch short of code change.
- **Remediation:** Read from configuration with sane default; ensure the value is logged at startup.

### S1.6 — DLQ topic derivation is fragile
- **Audit:** kafka
- **File:** `src/platform/host/adapters/KafkaOutboxPublisher.cs` (DLQ resolution path)
- **Violation:** DLQ topic name is computed by string replacement on the source topic name instead of routing through the canonical `TopicNameResolver`. Any topic that does not match the assumed naming pattern will silently misroute its DLQ traffic.
- **Remediation:** All DLQ resolution must go through the same resolver as live topics. Add a unit test that asserts every topic in `create-topics.sh` resolves to a valid DLQ pair.

## Strong positives (worth recording)

These came up across clusters and represent real architectural wins from the phase-1-gate work — record them so future audits don't second-guess them:

- **Middleware order is locked at construction** with `ValidateMandatoryDependencies()` throwing if any of the 8 stages is missing. No `Sort/Insert` mutations possible. (`RuntimeControlPlaneBuilder.cs:105-115`)

- **Optimistic concurrency** uses per-aggregate Postgres advisory lock + version check inside the transaction, *plus* the new unique index on `(aggregate_id, version)` as defense in depth.
- **Engine isolation verified by grep** — zero `SaveAsync` / `DbContext` / `Kafka*` / `IRepository` references anywhere under `src/engines/`.
- **Projections layer no longer references Runtime** — `Whycespace.Projections.csproj` cleanly broken; handlers consume `IEventEnvelope` from shared contracts only. (DG-R7-01 remediated.)
- **Projection hardening test actually exercises** replay safety, mutation isolation, and idempotency — not smoke. Defensive-copy semantics verified.
- **Domain layer purity** — zero external dependencies under `src/domain/`. WorkflowExecution aggregate is a clean event-sourced design with proper invariant enforcement *before* event emission.
- **HSID v2.1 deterministic ID engine** is fully compliant: no clock, no random, topology-bounded sequences, single-stamp-point.
- **Replay determinism sentinels** (`ExecutionHash="replay"`, `PolicyHash="replay"`, `Timestamp=MinValue`) intact and respected by the chain anchor pipeline.
- **WBSM architecture tests** (`tests/unit/architecture/WbsmArchitectureTests.cs`) pin 12 structural invariants programmatically. New types added by phase-1-gate are covered.

## New-rule candidates captured

The following rules were identified during this sweep as not currently covered by an existing guard. Files written under `claude/new-rules/`:

- `20260408-103326-determinism.md` — **DET-SEED-DERIVATION-01**: Seeds passed to `IIdGenerator.Generate(seed)` must derive only from stable command coordinates; never wall-clock, never RNG.
- `20260408-103326-engines.md` — **E-LIFECYCLE-FACTORY-CALL-SITE-01**: T1M engines are forbidden from invoking aggregate command methods directly; all state transitions must flow through a lifecycle event factory.
- `20260408-103326-activation.md` — **ACT-FABRIC-ROUNDTRIP-TEST-01**: Every domain event registered with the event fabric must have at least one integration test that round-trips it through the real fabric (not in-memory doubles), so activation drift cannot pass tests silently.
- `20260408-103326-tests.md` — **TEST-HARNESS-FABRIC-PARITY-01**: Integration tests that bypass the real event fabric must be explicitly tagged and cross-checked against an activation registry, to prevent the S0.1 class of failure.
