## 4. Informational notes (not findings)

1. **R-WF-PAYLOAD-REGISTRY-01 scope clarification.** The payload-type registry enforces discriminator stamping for workflow *lifecycle event* payloads (Started payload, StepCompleted output) so they round-trip typed through Postgres replay. `ApprovalDecisionPayload` is a *command* payload, not a lifecycle event payload — it is consumed by the dispatcher's approval branch, not written to the event stream. R3.A.6 does not add discriminator stamping for it; existing command serialization pipeline handles it. No action required.

2. **Engine-level AwaitingApproval direct test coverage is indirect.** Per the pre-existing `T1M-RESUME-TEST-COVERAGE-01` open item (tracked in `claude/new-rules/_archives/20260407-230000-workflow-resume-payload-and-test-coverage.md`), a `T1MWorkflowHarness` integration fixture is still missing. AwaitingApproval → Suspended emission is covered **indirectly** via `WorkflowApprovalProjectionTests` (which asserts the projection reducer against seeded Suspended events) and `WorkflowApprovalReplayServiceTests` (which asserts post-Suspended behaviour). Building the harness is out of R3.A.6 scope.

3. **Incidental test-infrastructure fix.** `RealInfraTestHost.cs` had pre-existing constructor-signature drift from `R-CHAIN-BREAKER-DELEGATION-01` (commit `29e73c2f`): `WhyceChainPostgresAdapter` and `RedisExecutionLockProvider` now require an `ICircuitBreaker` parameter. R3.A.6 incidentally fixed this by injecting `NoOpTestCircuitBreaker` so the integration-test project builds. The dedicated breaker-behaviour tests still cover breaker dynamics — the no-op is only used where breaker behaviour is not the test subject.

4. **D4 temporary carrier language.** Every doc/code reference to `human_approval_granted:` prefix on `WorkflowExecutionResumedEvent.PreviousFailureReason` is clearly marked as an approval-granted carrier (NOT failure semantics). Confirmed in: design note §1.5 row 1 + §2.4; factory XML doc on the new `Resumed(aggregate, approvalSignalOverride)` overload; guard rule `R-WF-APPROVAL-03`; audit entry for `R-WF-APPROVAL-03`.
