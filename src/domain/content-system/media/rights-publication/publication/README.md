# publication (SCAFFOLD — pending implementation)

## Purpose

The `publication` leaf owns **media publication lifecycle** — distinct from version activation. A media asset can have an active version without yet being published.

## Owns (planned)

- Publication identity, asset ref / version ref, audience scope, status (Unpublished / Published / Withdrawn), publication timestamps.
- Publish / republish / withdraw transitions.

SCAFFOLD only in P2.6.CS.10.
